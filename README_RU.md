## 💡 Чему вы здесь научитесь?

В этом проекте собраны практические примеры самых частых задач, с которыми сталкивается разработчик:
- **Обработка команд:** Как правильно реагировать на команды `/start` и `/help`.
- **Интерактивные клавиатуры:** Создание кнопок с callback-данными (обратным вызовом) и кнопок-ссылок на внешние сайты.
- **Современные коллбеки:** Обработка нажатий на кнопки без громоздких лямбда-функций с помощью магических фильтров (`F.data`).
- **Медиа и стикеры:** Отправка стикеров по их уникальному ID и чтение метаданных (ID файла, размер) из стикеров, которые присылает пользователь.
- **Безопасные фоллбеки (заглушки):** Обработка любых неожиданных сообщений от пользователя без вылета ошибки и падения бота.
- **Безопасность:** Правильное хранение секретных токенов в переменных окружения (`.env`), чтобы случайно не слить их в открытый доступ на GitHub.

## 🤖 Получение токена бота у BotFather

Каждому Telegram-боту нужен свой уникальный секретный ключ — **Токен (Token)**. Он позволяет серверам Telegram понять, какому именно боту предназначены сообщения.

1. Откройте Telegram и найдите официального бота **[@BotFather](https://t.me/BotFather)** (обязательно с синей галочкой!).
2. Запустите диалог и отправьте команду `/newbot`.
3. Следуйте инструкциям бота:
   - Введите **отображаемое имя** бота (например, `Мой Первый Бот`).
   - Введите **username (имя пользователя)** бота (обязательно должно заканчиваться на `bot`, например: `my_first_test_bot`).
4. В ответ BotFather пришлёт сообщение с длинной строкой, которая выглядит примерно так:
   `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
5. **Скопируйте этот токен!** Никому его не показывайте и никогда не выкладывайте в открытые чаты или репозитории.

## 📦 Установка и настройка проекта

### 1. Скачайте проект (Клонирование репозитория)
```bash
git clone https://github.com/Rifleks/basic_aiogram_bot.git
cd basic_aiogram_bot
```

### 2. Создайте виртуальное окружение
Виртуальное окружение (venv) изолирует библиотеки этого проекта от других программ на вашем компьютере. Это стандарт разработки на Python!

```bash
# Для Windows:
python -m venv venv
venv\Scripts\activate

# Для macOS / Linux:
python3 -m venv venv
source venv/bin/activate
```
*(После активации в начале строки вашего терминала должна появиться надпись `(venv)`).*

### 3. Установите необходимые библиотеки
Установим `aiogram` и `python-dotenv` из готового списка зависимостей:
```bash
pip install -r requirements.txt
```

### 4. Настройте переменные окружения
Никогда не вставляйте токен напрямую в код в файлах `.py`! Специальные роботы-парсеры на GitHub могут найти ваш токен за секунды и использовать бота в своих целях.
1. Найдите в папке проекта файл `.env.example`.
2. Переименуйте его в `.env` (или создайте копию с именем `.env`).
3. Откройте файл `.env` в текстовом редакторе и вставьте ваш токен от BotFather:
   ```env
   BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
   ```

### 5. Запустите бота

В проекте есть два варианта запуска:

#### Модульная версия (рекомендуется)

Это основной вариант проекта. Код разделён по папкам (`handlers`, `keyboards` и т.д.), как это обычно делают в реальных проектах.

```bash
python main.py
```

#### Версия в одном файле

Если вы хотите посмотреть, как выглядит тот же самый бот без разделения на модули, можно запустить файл `bot.py`. Он содержит тот же функционал, но весь код находится в одном файле, что удобно для изучения или быстрых экспериментов.

```bash
python bot.py
```

После запуска в консоли появится сообщение об успешном старте. Теперь откройте Telegram, найдите своего бота и отправьте команду `/start`.

## 🗺 Архитектура: один файл против модульного подхода

Когда вы только учитесь, писать всё в одном файле удобно. Но как только код бота переваливает за 200-300 строк, ориентироваться в нём становится невозможно. Мы приготовили два варианта архитектуры, чтобы вы поняли разницу!

### 1. Обычный прототип в одном файле (`bot.py`)
Идеально подходит для быстрых тестов и проверки гипотез. Здесь всё — диспетчер, хендлеры (обработчики) и клавиатуры — находится в одном скрипте. Если вы хотите просто посмотреть, как работает асинхронный код, начните с него.

### 2. Модульная архитектура (Рекомендуется!)
В реальных проектах код всегда разделяют на логические модули с помощью **Роутеров (Router)**. Роутер — это как «мини-диспетчер», который отвечает за отдельную часть функционала бота (например, только за регистрацию, или только за оплату).

```text
my_aiogram_bot/
│
├── handlers/            # Логика вашего бота (обработчики сообщений)
│   ├── __init__.py
│   ├── base.py          # Базовые команды (/start, /help) и коллбеки меню
│   └── media.py         # Обработка стикеров и эхо-хендлер (любый текст)
│
├── keyboards/           # Пользовательский интерфейс
│   ├── __init__.py
│   └── inline.py        # Функции для генерации клавиатур и кнопок
│
├── config.py            # Чтение и валидация токена из .env файла
├── main.py              # Точка входа: регистрирует роутеры и запускает бота
├── .env.example         # Пример файла с секретами для других разработчиков
├── .gitignore           # Говорит Git'у, какие файлы НЕЛЬЗЯ загружать в интернет (.env!)
└── requirements.txt     # Список зависимостей проекта
```

## 🧠 Важные концепции для новичков простым языком

### Что такое Long Polling и чем он отличается от Webhooks?
В этом проекте используется **Long Polling** (метод `dp.start_polling(bot)`). Это режим, при котором ваш скрипт сам непрерывно спрашивает сервер Telegram: *"Есть новые сообщения для моего бота?"*
Это лучший вариант для новичков, потому что он работает прямо на вашем домашнем компьютере или ноутбуке без необходимости настраивать белые IP-адреса, домены и SSL-сертификаты (которые обязательны для технологии Webhooks).

### Магические фильтры (`F`)
В старых версиях библиотек, чтобы проверить нажатие на кнопку или тип сообщения, приходилось писать сложные лямбда-функции. В aiogram 3 появились **Магические фильтры (Magic Filters)**, обозначаемые буквой `F`:
- `F.data == "get_commands_list"` — легко и красиво проверяет, что нажата кнопка с определённым `callback_data`.
- `F.sticker` — проверяет, что пришедшее сообщение является стикером.
Код становится читаемым даже для тех, кто не знает Python глубоко!

### Почему важен порядок регистрации хендлеров!
Диспетчер aiogram проверяет сообщения по цепочке хендлеров **строго сверху вниз**, в том порядке, в котором вы их зарегистрировали. Как только сообщение подошло под какой-то фильтр, оно обрабатывается, и дальше проверка останавливается!

Обратите внимание на файл `main.py`:
```python
dp.include_router(base.router)
dp.include_router(media.router)
```
Почему мы сначала подключаем `base.router`? Потому что внутри `media.py` у нас есть «фоллбек» (`@router.message()`) — хендлер без фильтров, который ловит **вообще любые** сообщения. Если бы мы подключили `media.router` первым, этот фоллбек перехватывал бы абсолютно всё, и до команд `/start` или `/help` из файла `base.py` очередь просто никогда бы не дошла!

## 🚀 Как запустить бота

Когда библиотеки установлены, а токен добавлен в файл `.env`, запустите команду в терминале:

```bash
python main.py
```
*(Или `python bot.py`, если вы хотите протестировать вариант в одном файле).*

В консоли появятся логи успешного запуска. Теперь откройте Telegram, найдите своего бота, нажмите **Запустить** (или отправьте `/start`) и радуйтесь результату! 🎉
